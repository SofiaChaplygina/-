Вариант 11 (Жадный алгоритм: раскраска графа):
Принцип работы алгоритма: Для каждой вершины проверяем цвета её соседей.
Назначаем минимальный возможный цвет, который не используется соседями.

Пошаговое выполнение жадного алгоритма раскраски графа:
1. Инициализация:
Создаем массив colors размером n (количество вершин)
Заполняем его значениями -1 (означает, что цвет не назначен)
Создаем пустой набор used для хранения цветов соседей
2. Основной цикл:
Перебираем каждую вершину графа по очереди
Для текущей вершины v:
Создаем множество used для хранения цветов соседей
Проверяем всех соседей вершины v
Если у соседа уже есть цвет (не -1), добавляем его в used
3. Назначение цвета:
Начинаем с цвета 0
Проверяем, есть ли этот цвет в множестве used
Если цвет есть в used, увеличиваем счетчик цвета на 1
Повторяем, пока не найдем цвет, которого нет в used
Назначаем найденный цвет текущей вершине
4. Завершение:
Повторяем шаги 2-3 для всех вершин
Возвращаем массив с назначенными цветами

Временная сложность:  O(V * E)

Объяснение временной сложности: 
Временная сложность алгоритма составляет O(V * E), потому что:
Алгоритм проходит по всем V вершинам графа
Для каждой вершины он анализирует все её рёбра (E)
Дополнительно требуется время на поиск свободного цвета среди соседей
Простыми словами: мы проверяем каждую вершину и для каждой смотрим на все её связи с другими вершинами, что в сумме даёт произведение количества вершин на количество рёбер. 
В худшем случае (для полного графа) это приводит к квадратичной сложности, но для разреженных графов алгоритм работает быстрее.

Контрольный вопрос 11(Имитация отжига):
Имитация отжига — метаэвристический алгоритм оптимизации, основанный на физическом процессе отжига металлов, где параметр температуры играет ключевую роль: 
при высокой температуре алгоритм активно исследует пространство решений, принимая даже худшие решения с определённой вероятностью (определяемой формулой P=exp(−ΔE/T)), а при низкой температуре происходит локальная оптимизация с принятием только лучших решений; 
процесс включает генерацию начального решения, последовательное создание новых вариантов и их оценку, при этом температура постепенно снижается до достижения минимума, что позволяет эффективно избегать локальных оптимумов, хотя и не гарантирует нахождение глобального решения.
